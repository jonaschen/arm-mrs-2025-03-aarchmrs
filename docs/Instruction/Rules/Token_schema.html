Instruction.Rules.Token object

A Token is a pattern-matching symbol that is present in the assembly stream. A Token is not a literal and it has no assembly or disassembly.

A Token has a pattern property that defines a regex for the symbols that it should match, and a default property that gives the rendering for the Token in a disassembled stream if no other rendering is available.

The four examples given below are for forms of COMMA, SPACE, UInteger, and EMPTY.

COMMA and SPACE are delimiters that occur in the assembly stream between assembly symbols.

UInteger matches any positive integer, starting from 0 and rising to infinity. The parent Rule can explicitly write the appropriate value in disassembly.

The EMPTY token states that an empty string is present. Some instances of Choice use this as the final option in a set of valid sets of symbols (if there is behavior that occurs when no value is present), whereas others end with a null.
